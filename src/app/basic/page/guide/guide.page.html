Component

file

devmonster에서 만든 FUT 시스템에 맞는 형태로 파일을 가공 및 활용할 수 있게 해주는 component 입니다.
back-end 와의 데이터 통신은 snake case 를 사용하기 때문에, file component 에서는 snake case 를 기본으로 사용합니다.
기본적으로 {{ '{ position:absolute, left:0, top: 0, width: 100%, height: 100%, opacity: 0 }' }} 인 레이아웃이기 때문에, 주로 button등과 함께 사용합니다.
여러개의 file component 에 같은 FutItem 리스트들을 물려서 사용할 수도 있습니다.
반드시 [view_type]을 정의하고 사용하여야 합니다.

examples
<app-button>
  file upload
  <app-file name="input" view_type="NOTICE"
  [value]=”form.list”
  [files]=”form.files”
  [file_json]=”form.file_json”
  ></app-file>
</app-button>
[multiple=false]
[type=MAIN] [multiple=true]
[type=SUB] [accept=camera]
[type=AVATAR] file.list[{{i}}] = {
    view_type: {{ item.view_type }},
    file_name: {{ item.file_name }},
    file_size: {{ item.file_size }}
} file.list.splice({{i}}, 1) 
list = {{ file.list | json }}
 
file_json = {{ file.file_json | json }}
Properties
Prop Type Description view_type string 업로드 시 FutItem에 사용할 view_type을 지정해 줍니다. multiple boolean default는 false. 해당 view_type의 파일을 여러개의 파일을 선택할 것 인지, 1개만 선택하게 할 것인지를 결정합니다.
false값을 가지면, 기존의 같은 view_type을 가진 FutItem은 새로운 파일을 업로드 시, 삭제 됩니다. accept string | 'camera' 일반적인 input[accept]와 동일하게 사용합니다.
camera속성을 사용할 경우, camera-service를 상속하고 있어서, 카메라 혹은 앨범에서 이미지를 선택할 수 있습니다. value FutItem[] 서버에서 반환하는 FutItem 형식의 리스트를 입력하거나, 반환합니다. files File[] 업로드할 파일들을 반환합니다. file_json File[] 업로드할 파일들을 반환합니다.
