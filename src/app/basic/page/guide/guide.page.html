Library Guide

Devmonster의 기본 프론트 개발 library guide 입니다. 모든 값의 디폴트 설정은 app.module.ts에서 설정을 하게 됩니다.

Animation

화면에 애니메이션을 쉽게 사용하기위해 미리 생성해둔, angular 애니메이션 셋 입니다.
basic/app.animation.ts에 정리되어 있습니다.

fadeAnimation

fade In Out 애니메이션 입니다.

example
<app-card @fadeAnimation></app-card>
fadeAnimation fadeAnimation

fadeInAnimation

fade In 애니메이션 입니다.

example
<app-card @fadeInAnimation></app-card>
fadeInAnimation fadeInAnimation

listAnimation

*ngFor와 같이 리스트에 애니메이션을 입히기 위해 사용 됩니다. In Out 애니메이션 입니다.

example
<div [@listAnimation]="listAnimationItemList.length">
    <app-card *ngFor=”let item of listAnimationItemList”>{{ item }}</app-card>
</div>
listAnimation
{{ item }}

listInAnimation

*ngFor와 같이 리스트에 애니메이션을 입히기 위해 사용 됩니다. In 애니메이션 입니다.

example
<div [@listInAnimation]="listInAnimationItemList.length">
    <app-card *ngFor=”let item of listInAnimationItemList”>{{ item }}</app-card>
</div>
listInAnimation
{{ item }}

bounceInAnimation

bounce In 애니메이션 입니다.

example
<app-card @bounceInAnimation></app-card>
bounceInAnimation bounceInAnimation

Component

화면에 tag를 직접 넣어 사용하거나, service와 연동하여, 새로운 화면을 띄워주는 역할을 합니다.
컨트롤 가능한 css variable은 각 component의 scss > :host의 가장 위에 있습니다.

Dialog

alert, modal 등 화면에 새로 띄우는 컴포넌트들 입니다. tag를 직접 넣어 사용하지 않고, service를 이용해서 사용합니다.

app-alertprivateservice

calendar.service로 사용합니다.

Example
this.alert.present({
    mode: 'devmonster',
    header: 'alert header',
    subHeader: 'alert sub header',
    message: 'alert message',
    buttons: [
        { text: 'cancel' },
        { text: 'submit', handler: () => {
            this.toast.present({ message: 'submit click' });
        }}
    ]
})
mode:devmonster mode:ionic
Properties
AlertOptions Prop Type Description mode 'devmonster' | 'ionic' | 'ios' | 'md' alert의 mode 입니다. header string alert의 header 입니다. subHeader string alert의 subHeader 입니다. message string alert의 message 입니다. inputs 
Array<{
    type?:string,
    name?:string,
    placeholder?:string,
    value?:string,
    label?:string,
    checked?:string,
    disabled?:string,
    id?:string,
    handler:(input: AlertInput) => void,
    min?:number, max?:number,
    cssClass?:string | string[],
    tabindex?:number
}}
alert의 input들 입니다. buttons 
Array<{ 
    text:string,
    handler?:(value:any)=>void
}}
alert의 buttons 입니다. input값을 받아와서 처리할 수 있습니다. cssClass string | string[] alert에 class를 추가합니다. backdropDismiss string alert의 backdrop을 클릭 시, alert을 닫을지 말지를 정합니다.

Input

기본적으로 form 안에서 사용하며, [name] 이 반드시 들어가야 합니다.
아래의 Properties를 공통으로 가집니다.

Properties
Prop Type Description value any component별로 다른 타입의 value를 가집니다.
각 component들의 properties에 정의되어 있습니다. readonly boolean 읽기 전용 input 상태 입니다. 클릭 이벤트를 막지는 않습니다. disabled boolean 사용불가 input 상태 입니다. 클릭 이벤트가 있으면, 같이 막힙니다. (change) Output value가 변할 때, 호출되는 Output 이벤트 입니다. value를 반환합니다.

calendar

달력으로 날짜를 고르는 인풋 입니다.
버튼 클릭 시, calendar-popover를 호출합니다.

example
<app-calendar name="calendar" label="calendar" [(ngModel)]=”form.date”></app-calendar>

~
properties
Prop Type Description value 'YYYY-MM-DD' 날짜포멧의 string을 값으로 가집니다. label string 캘린더 위에 띄우는 라벨입니다. min date string 선택할 수 있는 가장 빠른 날짜 입니다. max date string 선택할 수 있는 가장 늦은 날짜 입니다. timePicker boolean 시, 분, 초를 선택하는 timePicker를 사용할지 말지 결정합니다. type 'date' | 'week' 하루단위나 주단위로 달력을 선택할 수 있는 기능입니다.

calendar-popoverprivate

calendar에서 호출되는 달력 입니다.

check

checkbox component 입니다. 텍스트와 함께 혹은 단독으로 사용할 수있습니다.

example
<app-checkbox name="checkbox" [(ngModel)]=”form.check”>text</app-checkbox>
check with text
Properties
Prop Type Description value boolean check가 되면 true, 아니면 false를 반환합니다. color Color check가 되었을 때의 색상을 변경합니다. on any default는 true. check가 되었을 때의 값을 변경할 수 있습니다. off any default는 false. check가 되지 않은 상태의 값을 변경할 수 있습니다.

input

input component 입니다. 일반적인 input과 동일하게 사용합니다. 이미지나 텍스트와 조합하여 사용할 수 있습니다.

examples
<app-input name="input" label="input" [(ngModel)]=”form.input”>
  <app-img slot="end" src="assets/basic/img/search.png"></app-img>
</app-input>

app-button-footer

이것은 하단에 고정되는 footer 버튼을 쉽게 만들기위한 component 입니다. 페이지 하단을 참고해주세요.

app-card

앱 카드에 주로 사용되는 레이아웃 입니다. 내부 header, title, subtitle, content는 ionic 원본을 사용합니다.

ion-card-header ion-card-title ion-card-subtitle ion-card-content ion-note ion-card-header button:false

app-copy-right

페이지 하단의 로고 및 카피라이트를 표시하기 위한 component 입니다.

app-img

기존 img component 입니다. 다크모드 호환, 이미지 사이즈 조정 등을 하기위해 만들었습니다.
서버에서 받아오는 이미지는 dark mode가 없으므로, 일반 img tag를 사용해야 합니다.

app-logo

페이지 전체에 공유하는 '앱의 로고' 레이아웃 입니다.

app-profile

프로필 이미지를 표시하기 위한 component 입니다. app-input-file 을 상속하고 있어서, 이미지 추가, 삭제를 할 수 있습니다.

app-row / app-col

flex 레이아웃을 짜기 위한 component 입니다.

size: auto size: 200px size: 2 not sized

app-table
